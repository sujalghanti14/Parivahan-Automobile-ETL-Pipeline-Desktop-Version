# Parivahan ‚Äî ETL Pipeline

An automated ETL pipeline that collects, cleans, and stores **Maker-wise Fuel data** for 4-wheeler vehicle registrations across all Maharashtra RTOs from the [Vahan Dashboard](https://vahan.parivahan.gov.in/vahan4dashboard/).

## What Each Step Does

### 1. Extract ‚Äî `Extract_MH_4W.py`
- Uses **Playwright** to detect dynamic button/dropdown IDs from the Vahan portal
- Uses **Selenium** to automate the browser:
  - Selects Maharashtra state
  - Configures Y-Axis = Maker, X-Axis = Fuel
  - Loops through all 60 RTOs √ó selected months
  - Selects vehicle categories (Motor Car, Maxi Cab, Luxury Cab, Motor Cab, Vintage, LMV)
  - Downloads an Excel report for each combination
- Renames each file using the report title and saves it to `Data/`

### 2. Transform ‚Äî `Transform.py`
- Reads all `.xlsx` files in `Data/`
- Reshapes each file from **wide matrix** ‚Üí **long tabular** format
- Extracts **RTO name**, **month**, and **year** from the report title
- Drops blank rows and zero-count rows
- Merges all files into `consolidated_fuel_data.csv`

**Output columns:** `title | rto | brand | fuel | count | month | year`

### 3. Load ‚Äî `load.py`
- Reads `consolidated_fuel_data.csv`
- Uploads data to a **Supabase** table (`RTA_Maharashtra`) in batches of 500 rows

## Setup

### 1. Install dependencies

```bash
pip install -r requirement.txt
```

Dependencies: `pandas`, `selenium`, `playwright`, `supabase`, `python-dotenv`, `openpyxl`

Install Playwright browsers (first time only):
```bash
playwright install chromium
```

### 2. Configure environment variables

Create a `.env` file in the project root:

```env
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=your-anon-or-service-role-key
```

### 3. Create the Supabase table

Your `RTA_Maharashtra` table should have these columns:

| Column  | Type    |
|---------|---------|
| title   | text    |
| rto     | text    |
| brand   | text    |
| fuel    | text    |
| count   | integer |
| month   | text    |
| year    | text    |

## Running the Pipeline

### Run all 3 steps at once
```bash
python ETL/ETL_Pipeline.py
```

### Run individual steps
```bash
# Extract only
python ETL/Extract_MH_4W.py

# Transform only
python ETL/Transform.py

# Load only
python ETL/load.py
```

## Customising the Extract

To download only specific RTOs or months, call `run_extract()` with custom arguments:

```python
from Extract_MH_4W import run_extract

run_extract(
    rto_list=["PUNE - MH12( 25-JAN-2017 )", "NASHIK - MH15( 05-JAN-2017 )"],
    months=["JAN", "FEB", "MAR"]
)
```
